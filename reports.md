# Проект: Задачи по Программированию на практику отладки / профилирования

## Основная проблема
Студенты, изучающие программирование, часто сталкиваются с трудностями при отладке и профилировании своих приложений. Отсутсвие таких навыков часто приводит к увеличению количества ошибок и снижению качества кода. Основные проблемы, с которыми сталкиваются студенты следующие:

- Многие студенты не знают, как правильно использовать инструменты отладки;
  
- Студенты часто не осознают, как профилирование может помочь в выявлении узких мест в производительности их приложений;

- Студенты могут иметь теоретические знания, но недостаток практического опыта в использовании средств отладки и профилирования может привести приводить к неуверенности и ошибкам;

- Даже если студенты используют инструменты отладки и профилирования, они могут не знать, как правильно интерпретировать полученные данные и применять их для улучшения кода.

## Цель проекта

Цель данного проекта — разработать модуль, который поможет студентам на практике освоить средства отладки и профилирования. Модуль будет включать в себя:

- **Генератор задач**: Генератор задач на практику отладки и профилирования с настройками сложности генерации.

- **Автопроверка задач**: Автоматическая проверка задач.

## Требования к проекту

- **Общая обертка-модуль**: Модуль должен предоставлять единый доступ к генератору задач и автопроверке задач.

- **Интеграция в CodeRunner**: Модуль должен быть совместим с платформой CodeRunner.

- **Параметры для настройки сложности условий**: Модуль должен иметь гибкие настройки сложности генерируемых заданий.
  
- **Демо-курс**: Для демонстрации проекта должен быть предоставлен демо-курс на e.moevm.info, использующий данный модуль.

## Категории пользователей и их потребности

### Студенты

Студенты являются основными пользователями модуля. Для них важны следующие аспекты:

- **Доступ к материалам**: Студенты хотят иметь возможность получать необходимые практические задания.

- **Обратная связь**: Возможность получать своевременные комментарии по выполненным заданиям.

### Преподаватели

Преподаватели играют ключевую роль в организации и проведении учебного процесса. Для них важны следующие аспекты:

- **Управление заданиями**: Преподаватели хотят иметь интрумент настройки генерации заданий, а также выбирать различные типы задач. 

## Сценарии использования

### Основной сценарий:

1. **Выбор темы**
   - **Действие:** Пользователь выбирает тему для генерируемой задачи: отладка или профилирование;
   - **Результат:** Система отображает конкретные задачи по теме и доступные параметры для настройки;

2. **Настройка параметров генерации**
   - **Действие:** Пользователь настраивает индивидуальные параметры для каждой задачи;
   - **Результат:** Система сохраняет заданные настройки;

3. **Генерация задач с ошибками**
   - **Действие:** Пользователь инициирует процесс генерации задач;
   - **Результат:** Система генирирует задачу (задачи) с заданными ошибками и предоставляет ее пользователю;

4. **Изучение сгенерированных задач**
   - **Действие:** Пользователь просматривает сгенерированные задачи и анализирует ошибки;
   - **Результат:** Пользователь получает возможность работать с задачами;

5. **Отправка исправленных решений**
   - **Действие:** Пользователь отправляет свои исправленные решения на проверку;
   - **Результат:** Система проверяет решения и предоставляет обратную связь о правильности исправлений;

6. **Завершение работы**
   - **Действие:** Пользователь завершает работу с задачами или инициирует генерацию нового набора задач;
   - **Результат:** Система сохраняет текущие настройки или завершает сессию.

### Альтернативный сценарий:

1. **Ошибка при генерации задач**
   - **Действие:** При попытке сгенерировать задачи возникает ошибка (например, из-за некорректных параметров);
   - **Результат:** Пользователю выводится сообщение об ошибке с предложением исправить параметры;

2. **Завершение работы**
   - **Действие:** Пользователь завершает работу с модулем;
   - **Результат:** Система сохраняет текущие настройки или завершает сессию.

## Макет UI (программный интерфейс)

1. **Генератор задач**
Модуль представляет собой библиотеку, где генератор задач настраивается через параметры сложности, передаваемые в виде словаря или конфигурационного объекта. Параметры включают:

   - Тему задания: отладка (поиск и исправление ошибок) или профилирование (оптимизация производительности);

   - Типы ошибок: синтаксические, логические, ошибки времени выполнения (для отладки); узкие места в скорости выполнения, потреблении памяти (для профилирования);

   - Настройки сложности, влияющие, например, на размер кода, количество ошибок, вложенность логики, использование конкретных структур данных, ограничения на оптимизацию.

Генератор создает задачу, которая содержит:

   - Текстовое описание: формулировка задачи (что нужно исправить или оптимизировать);

   - Код с ошибками (для отладки) или неоптимизированный код (для профилирования);

   - Набор тестов: входные данные и ожидаемые результаты для проверки корректности;

   - Метрики производительности (для профилирования): например, допустимое время выполнения или лимит памяти.

Пример:
При настройке {"topic": "debugging", "mistakes_num": "2"} генератор создает функцию с 2 логическими ошибками, например, неправильное условие цикла или некорректное вычисление.

2. Система автопроверки
Автопроверка работает в двух режимах:

   - Для отладки: проверяет, что исправленный код студента проходит все тесты из условия задачи;

   - Для профилирования: анализирует, достигнуты ли целевые метрики производительности (например, сокращение времени выполнения на 30%).

Процесс проверки:

   - Запуск тестов: код студента выполняется на наборе тестов из задания;

   - Сравнение результатов: выходные данные сверяются с эталонными;

   - Профилирование (для задач на оптимизацию): замеряются время выполнения, использование памяти;

   Формирование отчета: какие тесты пройдены/не пройдены, насколько улучшена производительность.

Пример:
Если студент оптимизировал алгоритм сортировки, система сравнит время выполнения исходного и исправленного кода на больших данных и выдаст процент улучшения.

3. Интеграция с CodeRunner
Модуль встраивается в CodeRunner как внешняя библиотека. Рабочий процесс:

Для преподавателя:

   - В интерфейсе создания задания в CodeRunner выбирается тип задачи "Отладка/Профилирование";

   - Настраиваются параметры генерации;

   - Модуль автоматически генерирует условие задачи и эталонное решение, которое сохраняется в системе;

Для студента:

   - В интерфейсе CodeRunner отображается сгенерированный код с описанием задачи;

   - Студент редактирует код прямо в веб-интерфейсе (исправляет ошибки или оптимизирует его);

   - При нажатии "Отправить" код автоматически проверяется: запускаются тесты и (для профилирования) сравниваются метрики с исходными;

   - Студент получает результат: пройдены ли тесты, достигнуты ли целевые показатели производительности.

4. Особенности реализации
Динамическая генерация: Каждый студент может получить уникальный вариант задачи (например, с разными входными данными), чтобы предотвратить списывание.
